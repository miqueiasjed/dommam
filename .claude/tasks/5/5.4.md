# Task 5.4 – DownloadController + Rota autenticada

## Objetivo
Criar o `DownloadController` com o endpoint `GET /api/roteiros/{slug}/download` protegido pelo middleware `VerificarAssinaturaAtiva`, retornando o PDF com headers corretos para download direto no navegador.

## Tipo
componente-dominio

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/Download/DownloadController.php` → criar
- `backend/routes/api.php` → modificar (adicionar rota)

## Especificação

### O que fazer:

**1. Criar `DownloadController`:**

```php
namespace App\Http\Controllers\Download;

use App\Http\Controllers\BaseApiController;
use App\Models\Roteiro;
use App\Services\Download\DownloadService;
use Illuminate\Http\Request;
use Illuminate\Http\Response;

class DownloadController extends BaseApiController
{
    public function __construct(
        private readonly DownloadService $downloadService,
    ) {}

    public function baixar(Request $request, string $slug): Response
    {
        $roteiro = Roteiro::where('slug', $slug)
            ->publicados()
            ->firstOrFail();

        $email = $request->get('email_membro');
        $nome  = $request->get('nome_membro', $email); // fallback para email se nome não disponível
        $ip    = $request->ip();

        $pdf = $this->downloadService->baixar($roteiro, $email, $nome, $ip);

        $nomeArquivo = "roteiro-{$roteiro->slug}.pdf";

        return response($pdf, 200, [
            'Content-Type'        => 'application/pdf',
            'Content-Disposition' => "attachment; filename=\"{$nomeArquivo}\"",
            'Content-Length'      => strlen($pdf),
            'Cache-Control'       => 'no-store, no-cache, must-revalidate',
            'Pragma'              => 'no-cache',
        ]);
    }
}
```

**2. Adicionar rota em `routes/api.php`:**

Dentro do grupo de rotas de membro com middleware `VerificarAssinaturaAtiva`:
```php
Route::get('roteiros/{slug}/download', [DownloadController::class, 'baixar']);
```

### Regras arquiteturais:
- Controller deve ser "fino" — apenas chama o Service e formata a response
- `email_membro` e `nome_membro` vêm do middleware via `$request->merge()`
- Usar `firstOrFail()` — retorna 404 automaticamente se slug não encontrado ou não publicado
- Headers `Cache-Control: no-store` são obrigatórios — cada PDF é único por usuário

### Critérios de aceitação:
- [ ] Classe `DownloadController` criada em `App\Http\Controllers\Download\`
- [ ] Método `baixar()` retorna `Response` (não `JsonResponse`)
- [ ] Headers `Content-Type: application/pdf` e `Content-Disposition: attachment` presentes
- [ ] `Cache-Control: no-store` presente para impedir caching proxy
- [ ] Rota `GET /api/roteiros/{slug}/download` adicionada em `api.php` dentro do grupo do membro
- [ ] Rota protegida pelo middleware `VerificarAssinaturaAtiva`
- [ ] Roteiro não publicado retorna 404

## Teste esperado

- Request sem cookie de sessão → 401
- Request com sessão válida + slug existente → response com `Content-Type: application/pdf`
