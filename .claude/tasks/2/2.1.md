## Especificação

### Model Sessao (`backend/app/Models/Sessao.php`):
```php
namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\Builder;
use Carbon\Carbon;

class Sessao extends Model
{
    protected $table = 'sessions';
    protected $fillable = ['email', 'token_hash', 'ip_address', 'user_agent', 'expira_em', 'usado_em'];
    protected $casts = ['expira_em' => 'datetime', 'usado_em' => 'datetime'];

    // Scope: sessões ainda válidas (não usadas e não expiradas)
    public function scopeAtiva(Builder $query): Builder
    {
        return $query->whereNull('usado_em')->where('expira_em', '>', Carbon::now());
    }

    // Scope: sessões expiradas por tempo (independente de uso)
    public function scopeExpirada(Builder $query): Builder
    {
        return $query->where('expira_em', '<=', Carbon::now());
    }
}
```

### SolicitarMagicLinkRequest (`backend/app/Http/Requests/Auth/SolicitarMagicLinkRequest.php`):
- Regra: `email` obrigatório, string, max:255, formato e-mail válido
- Mensagem de erro em português no método `messages()`
- `authorize()` retorna `true`

### VerificarTokenRequest (`backend/app/Http/Requests/Auth/VerificarTokenRequest.php`):
- Regra: `token` obrigatório, string, tamanho exato de 64 caracteres (hex SHA-256)
- Mensagem de erro em português
- `authorize()` retorna `true`

### Regras visuais aplicáveis:
- Nomes de método em português (scopes: `ativa`, `expirada`)
- Mensagens de validação em português
- Sem lógica de negócio nos models — apenas scopes simples

### Critérios de aceitação:
- [ ] `Sessao::ativa()` filtra apenas registros com `usado_em IS NULL` e `expira_em > now()`
- [ ] `Sessao::expirada()` filtra registros com `expira_em <= now()`
- [ ] `SolicitarMagicLinkRequest` rejeita e-mail inválido com mensagem em português
- [ ] `VerificarTokenRequest` rejeita token com tamanho diferente de 64 chars
- [ ] Todos os arquivos têm namespace correto: `App\Models`, `App\Http\Requests\Auth`
