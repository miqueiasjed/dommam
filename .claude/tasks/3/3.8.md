# Task 3.8 – AdminUserSeeder

## Objetivo
Criar o seeder que inicializa o banco com um admin padrão (email + senha + sem TOTP por padrão) para setup inicial do projeto.

## Tipo
config

## Arquivos a criar/modificar
- `backend/database/seeders/AdminUserSeeder.php` → criar
- `backend/database/seeders/DatabaseSeeder.php` → modificar (chamar AdminUserSeeder)

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/AdminUser.php` → ver fillable e casts (Task 3.1)
- `backend/database/seeders/DatabaseSeeder.php` → ver estrutura atual
- `backend/database/migrations/2026_05_13_225130_create_admin_users_table.php` → confirmar colunas

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**AdminUserSeeder.php:**
- Namespace: `Database\Seeders`
- Método `run(): void`
  - Usa `AdminUser::firstOrCreate(['email' => 'admin@danuziohistory.com.br'], [...])`
  - Dados:
    - nome: 'Administrador'
    - email: 'admin@danuziohistory.com.br'
    - password: `Hash::make('admin123')` — senha fraca propositalmente para setup; comentar que deve ser trocada
    - role: 'admin'
    - ativo: true
    - totp_secret: null (2FA configurado via interface do admin)
  - `$this->command->info('Admin padrão criado/verificado: admin@danuziohistory.com.br');`

**DatabaseSeeder.php:**
- Adicionar chamada: `$this->call([AdminUserSeeder::class]);`

### Critérios de aceitação:
- [ ] `php artisan db:seed --class=AdminUserSeeder` roda sem erros
- [ ] Segundo `db:seed` não duplica o admin (firstOrCreate)
- [ ] Admin criado tem role = 'admin' e ativo = true
- [ ] `php artisan db:seed` (DatabaseSeeder) também executa AdminUserSeeder

## Teste esperado
- `php artisan db:seed --class=AdminUserSeeder`
- `php artisan tinker` → `AdminUser::where('email', 'admin@danuziohistory.com.br')->first()`
- Verificar que role = 'admin'
