# Task 3.4 – FileUploadService + AuditLogService

## Objetivo
Criar o serviço de upload de arquivos (capa JPG/PNG e PDF para Storage) e o serviço de auditoria (registra toda ação de CRUD de admins).

## Tipo
config

## Arquivos a criar/modificar
- `backend/app/Services/Admin/FileUploadService.php` → criar
- `backend/app/Services/Admin/AuditLogService.php` → criar

## Contexto necessário (arquivos para ler antes)

## Especificação

### O que fazer:

**FileUploadService.php:**
- Namespace: `App\Services\Admin`
- Método `uploadCapa(UploadedFile $arquivo): string`
  - Valida: mimes jpeg,jpg,png,webp | max 5MB
  - Gera nome único: `capas/{slug}-{uuid}.{ext}`
  - Armazena em disco configurado em `filesystems.default` (ou `public` em dev)
  - Retorna URL pública: `Storage::url($path)`
  - Lança `\InvalidArgumentException` se tipo inválido
- Método `uploadPdf(UploadedFile $arquivo): string`
  - Valida: mimes pdf | max 50MB
  - Gera nome único: `roteiros/{uuid}.pdf`
  - Armazena em disco privado (sem acesso público direto)
  - Retorna o `$path` (não URL pública — download será via marca d'água no Plano 5)
  - Lança `\InvalidArgumentException` se tipo inválido
- Método `remover(string $path): void`
  - Chama `Storage::delete($path)` se arquivo existir

**AuditLogService.php:**
- Namespace: `App\Services\Admin`
- Método `registrar(AdminUser $admin, string $acao, ?Model $entidade = null, ?array $dadosAntes = null, ?array $dadosDepois = null): AdminAuditLog`
  - Cria registro em `admin_audit_log`
  - `entidade_tipo`: nome da classe sem namespace (ex: `Roteiro`)
  - `entidade_id`: `$entidade->id`
  - `ip_address`: `request()->ip()`
  - Retorna o log criado
- Constantes de ações (como strings):
  - `ACAO_CRIAR = 'criar'`
  - `ACAO_ATUALIZAR = 'atualizar'`
  - `ACAO_DESPUBLICAR = 'despublicar'`
  - `ACAO_DELETAR = 'deletar'`
  - `ACAO_AGENDAR = 'agendar'`

### Critérios de aceitação:
- [ ] `FileUploadService::uploadCapa()` retorna URL string
- [ ] `FileUploadService::uploadPdf()` retorna path (não URL)
- [ ] `FileUploadService::remover()` chama Storage::delete sem erros
- [ ] `AuditLogService::registrar()` cria registro em admin_audit_log com ip_address
- [ ] Constantes de ação definidas e acessíveis via `AuditLogService::ACAO_CRIAR` etc.

## Teste esperado
- `php artisan tinker` → instanciar os services sem erros
- Verificar que `admin_audit_log` recebe registros ao chamar `registrar()`
