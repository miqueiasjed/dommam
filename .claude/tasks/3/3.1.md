## Especificação

### O que fazer:

**Migration adicional** — adicionar coluna `role` em `admin_users`:
```php
$table->enum('role', ['admin', 'operator'])->default('operator')->after('ativo');
```

**AdminUser.php:**
- Extende `Authenticatable`
- `$fillable`: nome, email, password, totp_secret, ativo, role, ultimo_acesso
- `$hidden`: password, totp_secret, remember_token
- `$casts`: ativo → boolean, ultimo_acesso → datetime
- Método `ehAdmin(): bool` → `$this->role === 'admin'`
- Método `ehOperator(): bool` → `$this->role === 'operator'`
- Relacionamento `auditLogs()` → hasMany(AdminAuditLog::class, 'admin_id')

**Roteiro.php:**
- Extende `Model`, usa `SoftDeletes`, `HasFactory`
- `$fillable`: titulo, slug, descricao, conteudo, capa_url, pdf_url, publicado_em, status, criado_por
- `$casts`: publicado_em → datetime
- Scopes:
  - `scopePublicados($q)` → where status = 'publicado'
  - `scopeAgendados($q)` → where status = 'agendado' + where publicado_em <= now()
  - `scopeRascunhos($q)` → where status = 'rascunho'
- Scope `scopePorStatus($q, $status)` → where status = $status (quando $status não nulo)
- Método `estaPublicado(): bool` → `$this->status === 'publicado'`
- Relacionamento `criador()` → belongsTo(AdminUser::class, 'criado_por')

**AdminAuditLog.php:**
- Extende `Model`, sem SoftDeletes
- `$fillable`: admin_id, acao, entidade_tipo, entidade_id, dados_antes, dados_depois, ip_address
- `$casts`: dados_antes → array, dados_depois → array
- Relacionamento `admin()` → belongsTo(AdminUser::class, 'admin_id')

### Critérios de aceitação:
- [ ] Migration `add_role_to_admin_users` cria coluna `role` enum('admin','operator') com default 'operator'
- [ ] `AdminUser::ehAdmin()` retorna true quando role = 'admin'
- [ ] `Roteiro::scopePublicados()` filtra por status = 'publicado'
- [ ] `Roteiro::scopeAgendados()` filtra por status = 'agendado' E publicado_em <= now()
- [ ] `AdminAuditLog::$casts` inclui dados_antes e dados_depois como array
- [ ] Todos os relacionamentos definidos corretamente
