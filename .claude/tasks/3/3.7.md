# Task 3.7 – Command PublicarRoteirosAgendados + Agendamento no Kernel

## Objetivo
Criar o Artisan command que publica automaticamente os roteiros agendados cujo `publicado_em <= now()`, e agendá-lo para rodar a cada minuto no Kernel (ou via scheduler do Laravel 12).

## Tipo
config

## Arquivos a criar/modificar
- `backend/app/Console/Commands/PublicarRoteirosAgendados.php` → criar
- `backend/routes/console.php` → modificar (adicionar agendamento — Laravel 12 usa console.php)

## Contexto necessário (arquivos para ler antes)
- `backend/app/Models/Roteiro.php` → ver scopes, especialmente `scopeAgendados()` (Task 3.1)
- `backend/routes/console.php` → ver como o projeto já usa agendamento
- `backend/bootstrap/app.php` → verificar se há registro do scheduler

## Skills necessárias
- laravel-arquitetura

## Especificação

### O que fazer:

**PublicarRoteirosAgendados.php:**
- Namespace: `App\Console\Commands`
- `$signature = 'roteiros:publicar-agendados'`
- `$description = 'Publica roteiros agendados cujo horário de publicação já chegou'`
- Método `handle(): int`
  - Busca roteiros: `Roteiro::where('status', 'agendado')->where('publicado_em', '<=', now())->get()`
  - Para cada roteiro:
    - Atualiza: `status = 'publicado'`
    - Não altera `publicado_em` (mantém o horário planejado)
    - `$roteiro->save()`
  - Loga quantidade: `$this->info("Publicados: {$roteiros->count()} roteiros")`
  - Retorna `Command::SUCCESS`

**console.php — adicionar agendamento:**

Laravel 12 usa `routes/console.php` com closure do scheduler:
```php
use Illuminate\Support\Facades\Schedule;

Schedule::command('roteiros:publicar-agendados')->everyMinute();
```

**Nota importante:** verificar se `bootstrap/app.php` já registra o scheduler via `withSchedule()`. Se não, adicionar:
```php
->withSchedule(function (Schedule $schedule) {
    $schedule->command('roteiros:publicar-agendados')->everyMinute();
})
```

### Critérios de aceitação:
- [ ] `php artisan roteiros:publicar-agendados` executa sem erros
- [ ] Roteiro com status='agendado' e publicado_em no passado é publicado ao rodar o command
- [ ] Roteiro com publicado_em no futuro NÃO é publicado
- [ ] `php artisan schedule:list` mostra o command com frequência 'Every minute'
- [ ] Log de "Publicados: X roteiros" aparece na saída do command

## Teste esperado

- Rodar `php artisan roteiros:publicar-agendados`
- Verificar que o roteiro agora tem `status = 'publicado'`
