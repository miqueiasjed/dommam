# Task 7.2 – Componente RoteiroCard

## Objetivo
Criar o `RoteiroCard` — card de poster (aspect ratio 2:3) com capa, título, data e tema; hover em desktop exibe sinopse e CTAs; mobile vai direto para `/roteiro/:slug`.

## Tipo
componente-dominio

## Arquivos a criar/modificar
- `frontend/src/components/roteiro/RoteiroCard.jsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/index.css` → tokens de cor disponíveis (bg-surface, text-muted etc.)
- `frontend/src/components/ui/Button.jsx` → ver props do Button para usar nos CTAs
- `frontend/src/components/ui/Tag.jsx` → ver props para exibir tag de tema

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

Criar `frontend/src/components/roteiro/RoteiroCard.jsx`:

**Props:**
```js
RoteiroCard({ roteiro })
// roteiro: { slug, titulo, sinopse, cover_image_url, publicado_em, tema, tipo }
```

**Estrutura visual:**
- Container: `relative aspect-[2/3] overflow-hidden rounded-lg cursor-pointer group`
- Imagem de capa: `object-cover w-full h-full` com `loading="lazy"`
  - Fallback: div cinza escuro se sem capa (`bg-zinc-900`)
- **Overlay de hover** (desktop, `group-hover`): 
  - Gradiente de baixo para cima: `bg-gradient-to-t from-black/90 via-black/50 to-transparent`
  - Título: `text-[13px] font-semibold text-white`
  - Data formatada: `text-[11px] text-muted` (ex: "Jan 2025")
  - Sinopse: até 2 linhas (`line-clamp-2`), `text-[11px] text-zinc-400`
  - Tag do tema (usar componente `Tag` existente)
  - Botão "Ver roteiro": `Button variante="primario" tamanho="sm"`
- **Mobile (sem hover):** exibir título + tag fixos no rodapé do card (sem sinopse/botão)
- Clique: navegar para `/roteiro/${roteiro.slug}` via `useNavigate`
- Animação de scale: `transition-transform duration-300 group-hover:scale-[1.03]` na imagem

**Formatação de data:**
```js
new Date(roteiro.publicado_em).toLocaleDateString('pt-BR', { month: 'short', year: 'numeric' })
```

### Regras visuais:
- Usar tokens CSS: `text-muted`, `bg-surface` — nunca hardcode de cor
- Exceção: gradientes com `from-black/90` são aceitos para overlay
- Tema escuro fixo

### Critérios de aceitação:
- [ ] Card renderiza capa com aspect-ratio 2:3
- [ ] Overlay aparece no hover em desktop (testar com `group-hover`)
- [ ] Título, data e tema visíveis no overlay
- [ ] Sinopse truncada em 2 linhas
- [ ] Clique navega para `/roteiro/:slug`
- [ ] Fallback visual quando `cover_image_url` é null
- [ ] Build sem erros: `npm run build`
