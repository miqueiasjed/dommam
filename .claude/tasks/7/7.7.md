### RoteiroCard.test.jsx

Mock de `react-router-dom` (useNavigate):
```js
vi.mock('react-router-dom', () => ({ useNavigate: () => vi.fn() }));
```

Fixture de roteiro:
```js
const roteiroFixture = {
  slug: 'teste-slug',
  titulo: 'Roteiro de Teste',
  sinopse: 'Sinopse curta',
  cover_image_url: 'https://example.com/capa.jpg',
  publicado_em: '2025-01-15',
  tema: 'História',
  tipo: 'Bastidor',
};
```

Testes:
1. Renderiza título do roteiro
2. Renderiza imagem de capa com `alt` correto
3. Renderiza tag do tema
4. Fallback: quando `cover_image_url` é null, não quebra

### RoteiroGrid.test.jsx

Testes:
1. Renderiza 8 skeletons quando `carregando: true`
2. Renderiza mensagem de erro quando `erro` é truthy
3. Renderiza EmptyState quando `roteiros: []` e `carregando: false`
4. Renderiza N cards quando `roteiros` tem itens (mock do RoteiroCard ou usar dados reais)

### BibliotecaPage.test.jsx

Mockar `useRoteiros`:
```js
vi.mock('../../hooks/useRoteiros', () => ({
  useRoteiros: () => ({
    roteiros: [],
    carregando: true,
    erro: null,
    filtros: { mes: '', temas: [], tipo: '' },
    setFiltros: vi.fn(),
    busca: '',
    setBusca: vi.fn(),
    ordenacao: 'mais_recentes',
    setOrdenacao: vi.fn(),
  }),
}));
```

Mockar `AppShell` para simplificar:
```js
vi.mock('../../components/layout/AppShell', () => ({ default: ({ children }) => <div>{children}</div> }));
```

Testes:
1. Página renderiza sem crash
2. SearchBar está presente no DOM
3. FilterBar está presente no DOM
4. RoteiroGrid está presente no DOM
