# Task 7.1 – Hook useRoteiros

## Objetivo
Criar o hook `useRoteiros` que busca `GET /api/roteiros` com suporte a filtros (mês, temas, tipo), busca textual com debounce 300ms e ordenação; gerencia estado de loading/error/dados.

## Tipo
hook

## Arquivos a criar/modificar
- `frontend/src/hooks/useRoteiros.js` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/hooks/useAuth.js` → ver padrão de hook existente
- `frontend/src/pages/BibliotecaPage.jsx` → entender como será consumido

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

Criar `frontend/src/hooks/useRoteiros.js` com:

```js
// Estado interno
const [roteiros, setRoteiros] = useState([]);
const [carregando, setCarregando] = useState(true);
const [erro, setErro] = useState(null);
const [filtros, setFiltros] = useState({ mes: '', temas: [], tipo: '' });
const [busca, setBusca] = useState('');
const [ordenacao, setOrdenacao] = useState('mais_recentes');
```

- `useEffect` que chama a API quando filtros/busca/ordenação mudam
- Debounce de 300ms aplicado somente na `busca` (usando `setTimeout`/`clearTimeout` dentro do effect)
- Montar os query params: `mes`, `temas` (array → `temas[]=x&temas[]=y`), `tipo`, `busca`, `ordenacao`
- Usar `fetch` com credenciais: `credentials: 'include'`
- A URL base vem de `import.meta.env.VITE_API_URL` (ex: `http://localhost:8000/api`)
- Tratar erro de rede e erro HTTP (status !== 200)
- Expor: `{ roteiros, carregando, erro, filtros, setFiltros, busca, setBusca, ordenacao, setOrdenacao }`

### Regras:
- Sem lógica visual — apenas gerenciamento de estado e fetch
- Variáveis em português
- Parâmetros de query em snake_case (convenção do backend Laravel)

### Critérios de aceitação:
- [ ] Hook retorna `roteiros` como array (vazio enquanto carrega)
- [ ] `carregando: true` durante o fetch, `false` após resposta
- [ ] Busca textual tem debounce de 300ms (não dispara request a cada tecla)
- [ ] Filtro de temas suporta múltipla seleção (array)
- [ ] Erro de rede popula o estado `erro`
- [ ] Build sem erros: `npm run build` (dentro de `frontend/`)
