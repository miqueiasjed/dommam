# Task 7.3 – Componente RoteiroGrid

## Objetivo
Criar o `RoteiroGrid` — grid responsivo de cards de roteiro (4 colunas desktop, 2-3 tablet, 1-2 mobile) com estado de loading via skeleton e estado vazio.

## Tipo
componente-dominio

## Arquivos a criar/modificar
- `frontend/src/components/roteiro/RoteiroGrid.jsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/components/roteiro/RoteiroCard.jsx` → entender props para renderizar
- `frontend/src/components/ui/EmptyState.jsx` → ver como usar estado vazio
- `frontend/src/index.css` → tokens de cor

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

Criar `frontend/src/components/roteiro/RoteiroGrid.jsx`:

**Props:**
```js
RoteiroGrid({ roteiros, carregando, erro })
```

**Estrutura:**
- Grid: `grid grid-cols-2 sm:grid-cols-3 md:grid-cols-4 gap-3 md:gap-4`

**Estado loading:** renderizar 8 skeleton cards:
```jsx
// Skeleton card
<div className="aspect-[2/3] rounded-lg bg-zinc-800/50 animate-pulse" />
```

**Estado erro:** 
```jsx
<p className="col-span-full text-center text-[13px] text-muted py-12">
  Erro ao carregar roteiros. Tente novamente.
</p>
```

**Estado vazio (roteiros.length === 0 e !carregando):**
- Usar componente `EmptyState` com:
  - `titulo="Nenhum roteiro encontrado"`
  - `descricao="Tente ajustar os filtros ou a busca."`
  - `icone={BookOpen}` (de lucide-react)
- Envolver em `<div className="col-span-full">`

**Estado normal:** mapear `roteiros` em `<RoteiroCard key={roteiro.slug} roteiro={roteiro} />`

### Regras visuais:
- Usar `animate-pulse` do Tailwind para skeleton
- Nunca hardcode de cor: usar tokens CSS ou classes Tailwind padrão do tema (zinc-800)
- Grid é a única fonte de verdade para o layout — sem flex alternativo

### Critérios de aceitação:
- [ ] Grid renderiza 4 colunas em desktop, 2 em mobile
- [ ] 8 skeletons aparecem durante `carregando: true`
- [ ] Mensagem de erro aparece quando `erro` é truthy
- [ ] EmptyState aparece quando `roteiros.length === 0` e não está carregando
- [ ] Cards renderizados quando `roteiros` tem itens
- [ ] Build sem erros: `npm run build`
