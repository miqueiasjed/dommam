## Especificação

### O que fazer:

**Estrutura geral:**
```jsx
export default function RoteiroPage() {
  const { slug } = useParams();
  const { roteiro, carregando, erro } = useRoteiro(slug);
  // renderização condicional
}
```

**Estado `carregando` — Skeleton:**
- Usar `AppShell` como wrapper
- Skeleton para o hero: `w-full h-[400px] bg-zinc-800/50 animate-pulse rounded-none`
- Skeleton para título: `h-8 w-2/3 bg-zinc-800/50 animate-pulse rounded` (margem `mt-8 mx-8`)
- Skeleton para texto: 3 linhas com `h-4 w-full/w-3/4/w-1/2 bg-zinc-800/50 animate-pulse rounded` (espaçadas com `gap-3`)

**Estado `erro === 'not_found'` — 404:**
- Usar `AppShell` como wrapper
- Centralizado: `flex flex-col items-center justify-center py-24 text-center px-8`
- Ícone `FileX` (lucide-react), tamanho 40, `text-zinc-600`
- "Roteiro não encontrado" (text-white, font-semibold, text-[18px])
- "O roteiro que você procura não existe ou foi removido." (text-muted, text-[13px], mt-2)
- Link para `/biblioteca`: `<Link>` com `Button variante="secundario"` — "← Voltar para a biblioteca"

**Estado `erro` (outros erros):**
- Usar `AppShell` como wrapper  
- Mesma estrutura mas com ícone `AlertCircle` e mensagem genérica de erro
- Botão "Tentar novamente" → recarrega a página (`window.location.reload()`)

**Estado de sucesso (roteiro carregado):**
```jsx
<AppShell>
  <div className="overflow-y-auto h-full"> {/* scroll interno */}
    <RoteiroHero
      capa_url={roteiro.capa_url}
      titulo={roteiro.titulo}
      data_publicacao={roteiro.data_publicacao}
      temas={roteiro.temas}
      instagram_url={roteiro.instagram_url}
    />

    <div className="max-w-3xl mx-auto px-8 py-8">
      {/* Botão de download */}
      <div className="mb-8">
        <DownloadButton slug={slug} titulo={roteiro.titulo} />
      </div>

      {/* Sinopse */}
      <div className="prose-custom">
        <h2 className="text-[13px] font-semibold uppercase tracking-widest text-muted mb-4">
          Sinopse
        </h2>
        {roteiro.sinopse?.split('\n\n').map((paragrafo, i) => (
          <p key={i} className="text-zinc-300 text-[14px] leading-relaxed mb-4">
            {paragrafo}
          </p>
        ))}
      </div>
    </div>

    {/* Navegação prev/next */}
    <RoteiroNav
      anterior={roteiro.anterior ?? null}
      proximo={roteiro.proximo ?? null}
    />
  </div>
</AppShell>
```

**Nota sobre scroll:** O `AppShell` tem `overflow-auto` no `<main>`. Garantir que a página faça scroll corretamente ao envolver em `<div className="overflow-y-auto h-full">`.

### Regras visuais aplicáveis:
- Usar tokens CSS: `bg-page`, `text-muted`
- Skeleton com `bg-zinc-800/50 animate-pulse`
- Tema escuro fixo
- Nunca hardcodar cores hex (exceto no gradiente do RoteiroHero que já é definido no próprio componente)

### Critérios de aceitação:
- [ ] Skeleton exibido enquanto `carregando === true`

- [ ] Botão "Baixar PDF" funcional (inicia download via `DownloadButton`)

- [ ] Navegação prev/next funcional via `RoteiroNav`
