# Task 8.6 – Componente `DownloadButton`

## Objetivo
Criar o botão de download de PDF que chama o endpoint `/api/roteiros/:slug/download`, recebe o arquivo como blob e dispara o download no browser com nome de arquivo correto.

## Tipo
componente-dominio

## Arquivos a criar/modificar
- `frontend/src/components/roteiro/DownloadButton.jsx` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/components/ui/Button.jsx` → props e como usar
- `frontend/src/utils/apiClient.js` → instância axios

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:

**Props:** `{ slug, titulo }`

**Estados internos:**
- `baixando`: boolean (false)
- `erro`: string | null

**Lógica de download:**
```js
async function handleDownload() {
  setBaixando(true);
  setErro(null);
  try {
    const response = await apiClient.get(`/api/roteiros/${slug}/download`, {
      responseType: 'blob',
    });
    // Criar URL temporária e disparar download
    const url = URL.createObjectURL(response.data);
    const link = document.createElement('a');
    link.href = url;
    link.download = `${titulo ?? slug}.pdf`;
    link.click();
    URL.revokeObjectURL(url);
  } catch (e) {
    setErro('Não foi possível baixar o PDF. Tente novamente.');
  } finally {
    setBaixando(false);
  }
}
```

**Renderização:**
- Usar componente `Button` com `variante="primario"` e `tamanho="lg"`
- Ícone `Download` do lucide-react quando idle
- Ícone `Loader2` com `animate-spin` + texto "Baixando..." quando `baixando === true`
- Botão desabilitado enquanto baixa
- Se `erro`: exibir mensagem de erro abaixo do botão:
  `<p className="mt-2 text-[11px] text-red-400">{erro}</p>`

**Estrutura de retorno:**
```jsx
<div className="flex flex-col items-start">
  <Button ...>
    <Download size={15} />
    {baixando ? 'Baixando...' : 'Baixar PDF'}
  </Button>
  {erro && <p ...>{erro}</p>}
</div>
```

### Regras visuais aplicáveis:
