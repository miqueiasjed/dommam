## Especificação

### O que fazer:
Substituir o stub atual de `LoginPage.jsx` pela implementação completa:

**Estados internos:**
- `estadoEnvio`: `'idle' | 'loading' | 'sucesso' | 'erro'`
- `email`: string do input
- `mensagemErro`: string | null

**Layout (tema escuro, centralizado na tela):**
- `bg-page` no container externo (`h-screen`, `flex`, `items-center`, `justify-center`)
- Card: `bg-surface`, `border border-zinc-800/80`, `rounded-xl`, `p-8`, `w-full max-w-sm`
- Logo/cabeçalho: ícone `BookOpen` do lucide-react + texto "Danuzio History" + badge "Backstage"
- Título: "Acesse o Backstage" (text-white, font-semibold)
- Subtítulo: "Insira seu e-mail para receber o link de acesso." (text-muted, text-[12px])

**Formulário:**
- Input de e-mail com label "E-mail"
  - `type="email"`, `required`, `autoFocus`
  - Estilo: `bg-surface-elevated border border-zinc-800/80 rounded-lg px-3 py-2 text-[13px] text-white w-full`
  - `focus:outline-none focus:ring-2 focus:ring-amber-500/50`
- Botão "Enviar link de acesso" (`variante="primario"`, `tamanho="lg"`, `type="submit"`, `w-full`)
  - Exibe `<Spinner />` + "Enviando..." quando `estadoEnvio === 'loading'`
  - `disabled` durante loading

**Estado `sucesso`:**
- Substituir o card inteiro por mensagem de confirmação:
  - Ícone `CheckCircle` (lucide-react, amber-500, tamanho 40)
  - "Link enviado!" (text-white, font-semibold)
  - "Verifique sua caixa de entrada. O link expira em 15 minutos." (text-muted)
  - Link "Tentar outro e-mail" → volta para `estadoEnvio = 'idle'`

**Estado `erro`:**
- Banner de erro acima do botão: `bg-red-900/20 border border-red-800/50 rounded-lg p-3`
- Texto: `mensagemErro` ou "Não foi possível enviar o link. Tente novamente."

**Lógica de submit:**
```js
async function handleSubmit(e) {
  e.preventDefault();
  setEstadoEnvio('loading');
  try {
    await apiClient.post('/api/auth/solicitar-link', { email });
    setEstadoEnvio('sucesso');
  } catch (err) {
    const msg = err.response?.data?.message ?? 'Não foi possível enviar o link.';
    setMensagemErro(msg);
    setEstadoEnvio('erro');
  }
}
```

### Regras visuais aplicáveis:
- Usar tokens CSS: `bg-page`, `bg-surface`, `bg-surface-elevated`, `text-muted`, `border-default`
- Nunca hardcodar cores hex
- Acento dourado: `amber-500/600` para foco e ícones
- Tema escuro fixo

### Critérios de aceitação:
- [ ] Estado idle: formulário exibido com campo e-mail + botão
- [ ] Estado loading: spinner no botão, input/botão desabilitados
- [ ] Estado sucesso: card substituído pela mensagem de confirmação
- [ ] Estado erro: banner de erro visível acima do botão, formulário ainda editável
- [ ] Link "Tentar outro e-mail" reseta o estado para idle

## Teste esperado
- Build sem erros: `npm run build` na pasta `frontend/`
- Verificação visual: acessar `/login` e verificar todos os estados (testar com e-mail inválido e válido)
