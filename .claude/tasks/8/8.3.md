# Task 8.3 – `MagicLinkCallbackPage` (implementação completa)

## Objetivo
Implementar a página de callback do magic link: lê o token da URL, chama a API para verificar, autentica o usuário e redireciona conforme o resultado.

## Tipo
pagina

## Arquivos a criar/modificar
- `frontend/src/pages/MagicLinkCallbackPage.jsx` → modificar (substituir stub)

## Contexto necessário (arquivos para ler antes)
- `frontend/src/contexts/AuthContext.jsx` → entender o método `login(email)` e estado `isAuthenticated`
- `frontend/src/hooks/useAuth.js` → como consumir o AuthContext
- `frontend/src/components/ui/Spinner.jsx` → spinner de carregamento
- `frontend/src/utils/apiClient.js` → instância axios

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:
Substituir o stub atual de `MagicLinkCallbackPage.jsx` pela implementação completa:

**Estados internos:**
- `estado`: `'verificando' | 'erro'` (sucesso não precisa de estado — redireciona imediatamente)
- `mensagemErro`: string | null

**Lógica (em `useEffect` no mount):**
```js
// 1. Ler token da URL: useSearchParams() → searchParams.get('token')
// 2. Se token ausente → setEstado('erro'), setMensagemErro('Link inválido.')
// 3. Chamar GET /api/auth/verificar com o token:
//    apiClient.get('/api/auth/verificar', { params: { token } })
// 4. Em caso de sucesso:
//    const { email, primeiro_acesso } = data
//    login(email)  // salva no AuthContext
//    navigate(primeiro_acesso ? '/boas-vindas' : '/biblioteca', { replace: true })
// 5. Em caso de erro:
//    setEstado('erro')
//    setMensagemErro(err.response?.data?.message ?? 'Link inválido ou expirado.')
```

**Layout estado `verificando` (tela cheia):**
- Centralizado na tela: `h-screen flex flex-col items-center justify-center gap-4 bg-page`
- `<Spinner tamanho={32} />`
- "Verificando link..." (text-muted, text-[13px])

**Layout estado `erro`:**
- Centralizado na tela com mesmo container
- Ícone `XCircle` (lucide-react, `text-red-400`, tamanho 40)
- "Link inválido ou expirado" (text-white, font-semibold, text-[16px])
- `mensagemErro` (text-muted, text-[13px])
- Botão (usando componente Button, variante="secundario"):
  - "Solicitar novo link" → navega para `/login`

### Regras visuais aplicáveis:
- Usar tokens CSS: `bg-page`, `text-muted`
- Nunca hardcodar cores hex
- Tema escuro fixo

### Critérios de aceitação:
- [ ] Com token ausente na URL: exibe estado de erro imediatamente
- [ ] Com token inválido (resposta de erro da API): exibe mensagem de erro + botão para `/login`
- [ ] Com token válido: chama `login(email)` e redireciona para `/boas-vindas` (primeiro acesso) ou `/biblioteca`
- [ ] Redirecionamento usa `replace: true` (não deixa a página de callback no histórico)

## Teste esperado
- Build sem erros: `npm run build` na pasta `frontend/`
- Verificação: acessar `/auth/verificar` sem token → estado de erro; com token de teste → verificando
