# Task 8.1 – Hook `useRoteiro`

## Objetivo
Criar o hook `useRoteiro(slug)` que busca um roteiro individual da API e retorna estado de carregamento, dados e erro.

## Tipo
hook

## Arquivos a criar/modificar
- `frontend/src/hooks/useRoteiro.js` → criar

## Contexto necessário (arquivos para ler antes)
- `frontend/src/hooks/useRoteiros.js` → padrão de hook existente (estados, fetch pattern)
- `frontend/src/utils/apiClient.js` → instância do axios configurada

## Skills necessárias
- frontend-design-system

## Especificação

### O que fazer:
Criar `frontend/src/hooks/useRoteiro.js` com as seguintes responsabilidades:

```js
export function useRoteiro(slug) {
  // estados: roteiro (null), carregando (true), erro (null)
  // useEffect dispara quando slug muda
  // chama GET /api/roteiros/:slug via apiClient
  // retorna { roteiro, carregando, erro }
}
```

- Se `slug` for falsy, não fazer fetch
- Em caso de erro 404, definir `erro` como `'not_found'` especificamente (para a RoteiroPage exibir mensagem diferenciada)
- Em outros erros HTTP, definir `erro` como a mensagem genérica
- Usar `apiClient` (já configurado com `withCredentials: true` e baseURL)

### Critérios de aceitação:
- [ ] Hook retorna `{ roteiro, carregando, erro }`
- [ ] `carregando` começa `true` e vai para `false` após o fetch
- [ ] Erro 404 da API resulta em `erro === 'not_found'`
- [ ] Outros erros resultam em string com mensagem de erro
- [ ] Novo fetch é disparado quando `slug` muda

## Teste esperado
- Build sem erros: `npm run build` na pasta `frontend/`
- Revisão de código: hook segue o mesmo padrão de `useRoteiros.js`
