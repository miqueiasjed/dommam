## Especificação

### Migration adicional (adicionar `event_id` à tabela `webhooks_log`):
```php
Schema::table('webhooks_log', function (Blueprint $table) {
    $table->string('event_id')->nullable()->unique()->after('id');
});
```

### WebhookLog.php:
- Extende `Model`
- `$table = 'webhooks_log'`
- `$fillable`: event_id, evento, payload, status, erro_mensagem, processado_em
- `$casts`: payload → array, processado_em → datetime
- Constantes de status:
  ```php
  const STATUS_PENDENTE    = 'pendente';
  const STATUS_PROCESSADO  = 'processado';
  const STATUS_ERRO        = 'erro';
  ```
- Método estático `jaProcessado(string $eventId): bool`:
  ```php
  return static::where('event_id', $eventId)
      ->whereIn('status', [self::STATUS_PROCESSADO])
      ->exists();
  ```
- Scope `scopePendentes($q)` → where status = 'pendente'

### Critérios de aceitação:
- [ ] Migration cria coluna `event_id` string nullable unique em `webhooks_log`
- [ ] `WebhookLog::jaProcessado('evt_123')` retorna false para event_id inexistente
- [ ] `$casts` inclui payload como array e processado_em como datetime
- [ ] `php artisan migrate` executa sem erros
