# Task 1.2 – Configurar CORS, Sanctum, Storage e Redis

## Objetivo
Configurar os serviços de infraestrutura do Laravel: CORS para aceitar o frontend, Sanctum para autenticação SPA via cookie httpOnly, filesystem storage (local + S3), e Redis para cache e filas.

## Tipo
config

## Arquivos a criar/modificar
- `backend/config/cors.php` → modificar
- `backend/config/sanctum.php` → modificar
- `backend/config/session.php` → modificar
- `backend/config/filesystems.php` → modificar
- `backend/config/cache.php` → modificar
- `backend/config/queue.php` → modificar

## Contexto necessário (arquivos para ler antes)
- `backend/config/cors.php` → ver configuração atual
- `backend/config/sanctum.php` → ver configuração atual
- `backend/config/session.php` → ver configuração atual
- `backend/.env.example` → ver variáveis disponíveis (criada na Task 1.1)

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

#### 1. CORS (`backend/config/cors.php`)
Publicar se não existir: `php artisan config:publish cors`

Atualizar para:
```php
'paths' => ['api/*', 'sanctum/csrf-cookie'],
'allowed_methods' => ['*'],
'allowed_origins' => [env('FRONTEND_URL', 'http://localhost:5173')],
'allowed_origins_patterns' => [],
'allowed_headers' => ['*'],
'exposed_headers' => [],
'max_age' => 0,
'supports_credentials' => true,  // obrigatório para cookies Sanctum
```

#### 2. Sanctum (`backend/config/sanctum.php`)
Publicar se não existir: `php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"`

Atualizar:
```php
'stateful' => explode(',', env('SANCTUM_STATEFUL_DOMAINS', sprintf(
    '%s%s',
    'localhost,localhost:3000,localhost:5173,127.0.0.1,127.0.0.1:8000,::1',
    Sanctum::currentApplicationUrlWithPort()
))),
'guard' => ['web'],
'expiration' => null,  // sem expiração automática (gerenciado por magic link)
```

#### 3. Session (`backend/config/session.php`)
Atualizar as chaves relevantes:
```php
'driver' => env('SESSION_DRIVER', 'database'),
'domain' => env('SESSION_DOMAIN', null),
'secure' => env('SESSION_SECURE_COOKIE', false),
'same_site' => 'lax',
'http_only' => true,
```

#### 4. Filesystem (`backend/config/filesystems.php`)
Garantir que o disco `s3` está configurado e adicionar disco `public` já configurado.
O disco `local` e `public` são padrão — verificar que estão presentes.
Para S3, manter as variáveis de env do `.env.example`.

#### 5. Cache (`backend/config/cache.php`)
```php
'default' => env('CACHE_DRIVER', 'redis'),
```
Garantir que o store `redis` usa `REDIS_HOST`, `REDIS_PASSWORD`, `REDIS_PORT` padrão.

#### 6. Queue (`backend/config/queue.php`)
```php
'default' => env('QUEUE_CONNECTION', 'redis'),
```
Garantir que a conexão `redis` está configurada com prefixo `danuzio_queue`.

#### 7. Executar `storage:link`
```bash
cd backend && php artisan storage:link
```

### Regras aplicáveis:
- `supports_credentials: true` no CORS é obrigatório para Sanctum funcionar com cookies
- Nunca colocar domínios de produção hardcoded — sempre via env
- O `SESSION_DOMAIN` deve ser configurável por env para funcionar em prod

### Critérios de aceitação:
- [ ] `config/cors.php` tem `supports_credentials => true` e usa `FRONTEND_URL`
- [ ] `config/sanctum.php` tem `SANCTUM_STATEFUL_DOMAINS` do env
- [ ] `config/session.php` tem `http_only => true` e `same_site => 'lax'`
- [ ] `config/cache.php` usa Redis como driver padrão
- [ ] `config/queue.php` usa Redis como driver padrão
- [ ] Link simbólico `backend/public/storage` criado com sucesso

## Teste esperado
- `cd backend && php artisan config:cache` → sem erros
- Verificar que `backend/public/storage` é um symlink para `backend/storage/app/public`
