# Task 1.1 – Instalar Laravel 12 e configurar .env.example

## Objetivo
Criar o diretório `backend/` com uma instalação limpa do Laravel 12 via Composer e personalizar o `.env.example` com todas as variáveis necessárias ao projeto.

## Tipo
config

## Arquivos a criar/modificar
- `backend/` → criar (projeto Laravel via composer create-project)
- `backend/.env.example` → modificar (adicionar variáveis do projeto)

## Contexto necessário (arquivos para ler antes)
- `CLAUDE.md` → variáveis de ambiente e stack necessária

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

1. Na raiz do projeto (`videos_ia/`), executar:
   ```bash
   composer create-project laravel/laravel:^12 backend
   ```

2. Após a instalação, substituir o conteúdo de `backend/.env.example` pelo seguinte (mantendo as variáveis padrão do Laravel e adicionando as do projeto):

```env
APP_NAME="Danuzio History Backstage"
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost:8000
FRONTEND_URL=http://localhost:5173

LOG_CHANNEL=stack
LOG_DEPRECATIONS_CHANNEL=null
LOG_LEVEL=debug

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=danuzio_backstage
DB_USERNAME=root
DB_PASSWORD=

BROADCAST_DRIVER=log
CACHE_DRIVER=redis
FILESYSTEM_DISK=local
QUEUE_CONNECTION=redis
SESSION_DRIVER=database
SESSION_LIFETIME=120
SESSION_DOMAIN=localhost
SESSION_SECURE_COOKIE=false

REDIS_HOST=127.0.0.1
REDIS_PASSWORD=null
REDIS_PORT=6379

MAIL_MAILER=resend
MAIL_HOST=smtp.resend.com
MAIL_PORT=465
MAIL_USERNAME=resend
MAIL_PASSWORD=
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS="noreply@danuziohistory.com.br"
MAIL_FROM_NAME="${APP_NAME}"

# Resend API (preferencial ao SMTP)
RESEND_API_KEY=

# Lastlink
LASTLINK_WEBHOOK_SECRET=
LASTLINK_API_URL=https://lastlink.com/api/v1
LASTLINK_API_TOKEN=

# Storage S3 (opcional — usar local em dev)
AWS_ACCESS_KEY_ID=
AWS_SECRET_ACCESS_KEY=
AWS_DEFAULT_REGION=us-east-1
AWS_BUCKET=
AWS_USE_PATH_STYLE_ENDPOINT=false

# Sanctum SPA
SANCTUM_STATEFUL_DOMAINS=localhost:5173,localhost:3000
```

3. Copiar `.env.example` para `.env`:
   ```bash
   cd backend && cp .env.example .env && php artisan key:generate
   ```

4. Verificar que `php artisan` executa sem erros.

### Regras aplicáveis:
- Não criar Models, Controllers ou Routes nesta task — apenas infraestrutura base
- Manter o `.env.example` com todos os valores em branco ou com valores de desenvolvimento local

### Critérios de aceitação:
- [ ] `backend/` existe com estrutura completa do Laravel 12
- [ ] `backend/.env.example` contém todas as variáveis listadas acima
- [ ] `php artisan` executa sem erros fatais a partir de `backend/`
- [ ] APP_KEY gerada no `.env` local

## Teste esperado
- Executar `cd backend && php artisan --version` → deve retornar `Laravel Framework 12.x.x`
- Executar `cd backend && php artisan config:clear` → sem erros
