# Task 1.4 – Migrations: roteiros e sessions

## Objetivo
Criar as migrations das duas tabelas centrais do domínio: `roteiros` (conteúdo histórico) e `sessions` (sessões de magic link dos membros).

## Tipo
config

## Arquivos a criar/modificar
- `backend/database/migrations/xxxx_create_roteiros_table.php` → criar
- `backend/database/migrations/xxxx_create_sessions_table.php` → criar (tabela custom de magic link)

## Contexto necessário (arquivos para ler antes)
- `backend/database/migrations/` → listar migrations existentes para evitar conflito de nome com sessões padrão do Laravel

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

#### ATENÇÃO — Conflito de nome `sessions`
O Laravel pode já ter uma migration de `sessions` padrão. Se existir, **deletá-la** antes de criar a nossa, pois a tabela custom substitui a padrão.

Verificar com:
```bash
ls backend/database/migrations/ | grep session
```
Se encontrar `xxxx_create_sessions_table.php` padrão do Laravel → deletar.

#### Migration 1: roteiros
```php
// Criar via: php artisan make:migration create_roteiros_table --create=roteiros

Schema::create('roteiros', function (Blueprint $table) {
    $table->id();
    $table->string('titulo');
    $table->string('slug')->unique();
    $table->text('descricao')->nullable();
    $table->longText('conteudo')->nullable();
    $table->string('capa_url')->nullable();
    $table->string('pdf_url')->nullable();
    $table->timestamp('publicado_em')->nullable();
    $table->enum('status', ['rascunho', 'agendado', 'publicado'])->default('rascunho');
    $table->unsignedBigInteger('criado_por')->nullable();
    $table->timestamps();
    $table->softDeletes();

    $table->index('status');
    $table->index('publicado_em');
    $table->index('slug');
});
```

#### Migration 2: sessions (magic link — tabela custom)
```php
// Criar via: php artisan make:migration create_sessions_table --create=sessions

Schema::create('sessions', function (Blueprint $table) {
    $table->id();
    $table->string('email');
    $table->string('token_hash', 64)->unique();
    $table->string('ip_address', 45)->nullable();
    $table->text('user_agent')->nullable();
    $table->timestamp('expira_em');
    $table->timestamp('usado_em')->nullable();
    $table->timestamps();

    $table->index('email');
    $table->index('token_hash');
    $table->index('expira_em');
});
```

### Regras aplicáveis:
- Colunas com semântica de negócio em **português** (ex: `expira_em`, `usado_em`, `publicado_em`)
- Colunas técnicas do Laravel permanecem em inglês (`timestamps()`, `softDeletes()`)
- Nunca guardar e-mail de assinante como FK — Lastlink é a fonte de verdade

### Critérios de aceitação:
- [ ] Migration `roteiros` criada com todos os campos especificados
- [ ] Migration `sessions` criada com todos os campos especificados
- [ ] Nenhum conflito com migration de `sessions` padrão do Laravel
- [ ] `php artisan migrate` executa sem erros nas duas migrations
