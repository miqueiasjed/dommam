# Task 1.3 – Criar BaseApiController com helpers de resposta

## Objetivo
Criar o `BaseApiController` que todos os controllers da API irão estender, com três helpers padronizados de resposta JSON: `success()`, `created()` e `error()`.

## Tipo
componente-dominio

## Arquivos a criar/modificar
- `backend/app/Http/Controllers/BaseApiController.php` → criar

## Contexto necessário (arquivos para ler antes)
- `backend/app/Http/Controllers/Controller.php` → ver classe base do Laravel

## Skills necessárias
- `laravel-arquitetura`

## Especificação

### O que fazer:

Criar `backend/app/Http/Controllers/BaseApiController.php`:

```php
<?php

namespace App\Http\Controllers;

use Illuminate\Http\JsonResponse;

abstract class BaseApiController extends Controller
{
    protected function success(mixed $dados = null, string $mensagem = 'Operação realizada com sucesso', int $status = 200): JsonResponse
    {
        $resposta = ['sucesso' => true, 'mensagem' => $mensagem];

        if (!is_null($dados)) {
            $resposta['dados'] = $dados;
        }

        return response()->json($resposta, $status);
    }

    protected function created(mixed $dados = null, string $mensagem = 'Recurso criado com sucesso'): JsonResponse
    {
        return $this->success($dados, $mensagem, 201);
    }

    protected function error(string $mensagem = 'Erro interno do servidor', int $status = 500, mixed $erros = null): JsonResponse
    {
        $resposta = ['sucesso' => false, 'mensagem' => $mensagem];

        if (!is_null($erros)) {
            $resposta['erros'] = $erros;
        }

        return response()->json($resposta, $status);
    }
}
```

### Regras aplicáveis:
- Métodos e variáveis em **português** (convenção do projeto)
- Classe abstrata — não deve ser instanciada diretamente
- Todos os controllers de API futuros devem estender `BaseApiController`, não `Controller`

### Critérios de aceitação:
- [ ] Arquivo criado em `backend/app/Http/Controllers/BaseApiController.php`
- [ ] Três métodos presentes: `success()`, `created()`, `error()`
- [ ] Classe declarada como `abstract`
- [ ] Métodos usam nomes em português para parâmetros e variáveis internas

## Teste esperado
- `cd backend && php artisan` → sem erros de autoload
- Verificar que a classe carrega: `php artisan tinker` → `new \App\Http\Controllers\BaseApiController` deve falhar com "Cannot instantiate abstract class" (comportamento correto)
