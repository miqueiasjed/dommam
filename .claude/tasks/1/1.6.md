# Task 1.6 – Mover frontend de src/ para frontend/

## Objetivo
Reestruturar o código React existente (atualmente em `src/` na raiz) para a pasta `frontend/`, ajustando todos os paths de configuração para que `npm run dev` e `npm run build` funcionem a partir de `frontend/`.

## Tipo
config

## Arquivos a criar/modificar
- `frontend/` → criar (mover conteúdo da raiz)
- `frontend/vite.config.js` → criar/ajustar paths
- `frontend/package.json` → criar (mover da raiz)
- `frontend/index.html` → criar (mover da raiz)

## Contexto necessário (arquivos para ler antes)
- `vite.config.js` → ver configuração atual de paths
- `package.json` → ver scripts e dependências atuais
- `index.html` → ver estrutura atual

## Skills necessárias
- `frontend-design-system`

## Especificação

### O que fazer:

#### 1. Criar estrutura de pastas
```
frontend/
├── src/
│   ├── components/
│   │   ├── ui/
│   │   ├── layout/
│   │   └── roteiro/
│   ├── hooks/
│   ├── utils/
│   └── pages/
├── public/
├── package.json
├── vite.config.js
├── index.html
└── eslint.config.js
```

#### 2. Mover arquivos da raiz para frontend/
Mover (não copiar) os seguintes itens:
- `src/` → `frontend/src/`
- `public/` → `frontend/public/`
- `index.html` → `frontend/index.html`
- `package.json` → `frontend/package.json`
- `package-lock.json` → `frontend/package-lock.json`
- `vite.config.js` → `frontend/vite.config.js`
- `eslint.config.js` → `frontend/eslint.config.js`
- `node_modules/` → **NÃO mover** — deletar da raiz e reinstalar em `frontend/`

#### 3. Ajustar vite.config.js
Verificar se há paths relativos que precisam ser atualizados. O `vite.config.js` típico não precisa de alteração quando o arquivo está na raiz do projeto frontend.

Garantir que o arquivo inclui:
```js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 5173,
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
      },
    },
  },
})
```

**Nota:** Adicionar o proxy `/api` → backend Laravel (porta 8000) é novo — necessário para comunicação em desenvolvimento.

#### 4. Instalar dependências em frontend/
```bash
cd frontend && npm install
```

#### 5. Reorganizar imports no src/ (se necessário)
Se o código em `src/` usa imports absolutos da raiz (ex: `import X from '/src/...'`), ajustar para relativos.

### Regras aplicáveis:
- Tema escuro fixo — sem alterações visuais nesta task
- Esta task é de reorganização estrutural — não adicionar features
- O proxy para a API Laravel deve usar porta 8000 (padrão `php artisan serve`)

### Critérios de aceitação:
- [ ] `frontend/src/` contém todo o código React existente
- [ ] `frontend/package.json` existe com todas as dependências
- [ ] `frontend/vite.config.js` tem proxy configurado para `http://localhost:8000`
- [ ] `cd frontend && npm run dev` inicia sem erros na porta 5173
- [ ] `cd frontend && npm run build` compila sem erros
- [ ] Raiz do projeto (`videos_ia/`) não tem mais `src/`, `node_modules/`, `package.json`, `index.html` soltos

## Teste esperado
- `cd frontend && npm run dev` → app abre em `http://localhost:5173` sem erros no console
- `cd frontend && npm run build` → `dist/` gerado sem erros
