## RoteiroFormPage — Lógica
```js
// Modo: criar (sem id na URL) ou editar (com /roteiros/:id/editar)
// Se editar: GET /api/admin/roteiros/{id} para pré-popular
// Submit criar: POST /api/admin/roteiros (multipart/form-data)
// Submit editar: PATCH /api/admin/roteiros/{id}
// Upload capa: POST /api/admin/roteiros/{id}/cover
// Upload PDF: POST /api/admin/roteiros/{id}/pdf
// Auto-slug: ao digitar título, gera slug (lowercase, hifens, sem acento)
// Auto available_at: published_at - 1 hora
```

## RoteiroFormPage — Visual

### Estrutura 2 colunas (desktop) / 1 coluna (mobile)
```
Coluna principal (flex-1):
  - Título (input grande)
  - Slug (input menor, editável)
  - Sinopse (textarea 4 linhas)
  - Temas (input tags — digita e pressiona Enter)

Coluna lateral (w-80):
  - UploadField capa (imagem)
  - UploadField PDF
  - Data publicação Instagram (datetime-local)
  - Data disponibilização portal (datetime-local, auto-sugerida)
  - Botões: [Salvar rascunho] [Publicar]
```

### Visual
- Page header: `← Novo roteiro` ou `← Editar: {título}`
- Inputs: `bg-[#111214] border border-amber-500/15 rounded-lg px-3 py-2 text-[13px] text-white focus:ring-2 focus:ring-amber-500/40 focus:outline-none`
- Labels: `font-mono text-[10px] uppercase tracking-widest text-zinc-500 mb-1.5`
- Tags: pill com `bg-amber-500/10 text-amber-400 border border-amber-500/20`
- Botão Salvar rascunho: outline zinc
- Botão Publicar: `bg-amber-500 text-black font-mono font-semibold` + modal de confirmação
