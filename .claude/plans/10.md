# Plano 10 – Polimento, E-mails de Onboarding e Lançamento

## Objetivo
Finalizar a sequência completa de comunicação por e-mail, popular a biblioteca com o acervo histórico, configurar analytics, realizar testes E2E e lançar o produto.

## Contexto de Negócio
Sprint final antes do lançamento. O fluxo de onboarding (D1, D3, D7, D25) retém assinantes após a compra. O seed de 10-15 roteiros garante que o produto já tem valor imediato. GA4 e Meta Pixel são obrigatórios para medir conversão. Testes E2E validam os fluxos críticos antes de exposição pública.

## Escopo

### Inclui

**E-mails de onboarding (backend):**
- `OnboardingJob`: processado em D+1, D+3, D+7 após `Purchase_Order_Confirmed`
  - D1: tour rápido (link para vídeo de 90s)
  - D3: 3 roteiros recomendados do arquivo (busca via `RoteiroService`)
  - D7: pedido de feedback (resposta por e-mail)
- `RetencaoJob`: D25 do ciclo — verifica roteiros não acessados (via tabela `downloads`) e envia e-mail
- `NovoRoteiroJob`: disparado quando `is_published` vira `true` — envia e-mail para todos os assinantes ativos (busca via Lastlink API)
- Templates Blade: `onboarding-d1`, `onboarding-d3`, `onboarding-d7`, `retencao-d25`, `novo-roteiro`
- Scheduler: agendar os Jobs de onboarding no momento certo

**Integração landing page:**
- Plugar URLs de checkout da Lastlink nos dois botões da landing (`/` ou arquivo HTML já entregue)
- Adicionar snippet de GA4
- Adicionar Meta Pixel

**Seed de roteiros históricos:**
- Seeder com 10-15 roteiros históricos do arquivo de Danuzio
  - Dados: título, slug, sinopse, temas, `published_at`, `available_at`, `is_published=true`
  - Capas e PDFs: upload manual via painel admin após seed (ou seeder aponta para URLs estáticas de teste)

**Monitoramento:**
- Configurar Sentry (ou Flare para Laravel) para erros em produção
- Alertas: job da fila falhando, endpoint de webhook retornando != 200, command `roteiros:publicar-agendados` não rodando

**Testes E2E:**
- Fluxo completo de compra simulada: webhook → e-mail boas-vindas → login → biblioteca → download
- Fluxo de admin: login+2FA → criar roteiro → agendar → verificar publicação automática
- Fluxo de cancelamento: webhook → sessão invalidada → acesso negado

**Checklist de lançamento:**
- [ ] Variáveis de ambiente de produção configuradas
- [ ] HTTPS configurado
- [ ] DNS de backstage.danuzio.com.br apontando corretamente
- [ ] Webhook URL registrada na Lastlink
- [ ] Secret HMAC configurado e testado
- [ ] Queue worker rodando em produção (Supervisor ou similar)
- [ ] Cron configurado em produção
- [ ] Pré-lançamento com 30-50 seguidores selecionados por Danuzio

### Não inclui (reservado para v2)
- Painel "minha conta" para assinante
- Embed de vídeos do Instagram
- Sistema de favoritos e listas pessoais
