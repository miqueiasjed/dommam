# Plano 8 – Frontend: Autenticação + Página Individual de Roteiro

## Objetivo
Implementar o fluxo completo de autenticação do membro (magic link UI, callback, tour de boas-vindas) e a página individual de roteiro com download de PDF.

## Contexto de Negócio
O fluxo de login é: usuário informa e-mail → recebe link por e-mail → clica → cai no portal autenticado. No primeiro acesso, vê um tour de boas-vindas de 60-90s. A página de roteiro exibe a capa em hero full-width, sinopse, link para o Instagram e o botão de download (que chama o endpoint do Plano 5, que gera PDF com marca d'água).

## Escopo

### Inclui

**Autenticação:**
- `LoginPage`: formulário com campo de e-mail + botão "Enviar link de acesso"
  - Estado: idle → loading → sucesso ("Verifique seu e-mail") → erro
  - Chama `POST /api/auth/solicitar-link`
- `MagicLinkCallbackPage`: rota `/auth/verificar?token=X`
  - Chama `GET /api/auth/verificar` → salva estado no `AuthContext` → redireciona
  - Estados: verificando, sucesso (redireciona para /biblioteca ou /boas-vindas), erro (token inválido/expirado)
- `WelcomeTourPage`: tela de boas-vindas para primeiro acesso
  - Tour de 3-4 slides: "O que é o Backstage", "Como funciona", "Acesse a biblioteca"
  - Botão "Começar" redireciona para /biblioteca
  - Flag no localStorage para não exibir novamente

**Página de Roteiro:**
- `RoteiroPage`: rota `/roteiro/:slug`
  - Chama `GET /api/roteiros/:slug`
  - Hero: capa cinematográfica em tela cheia com gradiente inferior
  - Título, data formatada (ex: "13 de maio de 2026"), tag(s) de tema
  - Link clicável "Ver no Instagram" (ícone + URL)
  - Sinopse em 2-3 parágrafos
  - Botão "Baixar PDF" → chama `GET /api/roteiros/:slug/download`, inicia download do arquivo
  - Navegação: "← Roteiro anterior" / "Próximo roteiro →"
  - Loading skeleton enquanto carrega
  - Estado de erro quando roteiro não encontrado (404 com link para /biblioteca)
- Hook `useRoteiro(slug)`: busca roteiro individual da API

### Não inclui (fica para outro plano)
- Admin panel UI → Plano 9
- Sequência de onboarding via e-mail → Plano 10
