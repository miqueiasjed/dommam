# Plano 3 – Admin Backend: Auth + CRUD de Roteiros

## Objetivo
Implementar o painel administrativo backend: autenticação de admins (login + senha + 2FA TOTP), CRUD completo de roteiros com upload de arquivos, agendamento automático de publicação e audit log.

## Contexto de Negócio
Admins usam login+senha+TOTP (não magic link). Dois roles: `admin` (tudo) e `operator` (CRUD roteiros, sem gerenciar admins). Todo CRUD de roteiros é registrado no `admin_audit_log`. O agendamento automático muda `is_published=true` quando `available_at <= now()` — isso é a promessa central do produto (roteiro 1h antes da publicação).

## Escopo

### Inclui
- Model `AdminUser` com roles (`admin`, `operator`)
- Model `Roteiro` com scopes: `publicados()`, `agendados()`, `rascunhos()`
- Model `AdminAuditLog`
- `AdminAuthService`: login (verifica senha + gera código TOTP), verify 2FA
- `AdminAuthController`: `POST /admin/login`, `POST /admin/2fa/verificar`
- Middleware `AutenticadoComoAdmin`
- `RoteiroService`: criar, atualizar, despublicar, agendar, listar com filtros
- `AdminRoteiroController`: CRUD completo (thin controller)
- `FileUploadService`: upload de capa (imagem) e PDF para Storage; retorna URL
- `AuditLogService`: registra ação, admin_user_id, target_id e detalhes
- `FormRequests`: `LoginAdminRequest`, `CriarRoteiroRequest`, `AtualizarRoteiroRequest`
- Command `PublicarRoteirosAgendados`: processa roteiros com `available_at <= now()` e `is_published=false`
- Agendamento do command no Kernel (a cada minuto)
- Seeder de admin user para setup inicial

### Não inclui (fica para outro plano)
- Frontend do painel admin → Plano 9
- Visualização de assinantes Lastlink → Plano 9
