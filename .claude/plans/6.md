# Plano 6 – Frontend: Design System e Estrutura Base

## Objetivo
Organizar o frontend no novo path `frontend/`, adaptar o design system para a identidade Danuzio History Backstage (cinematográfico, preto e dourado), e criar os componentes base reutilizáveis e a estrutura de roteamento com React Router.

## Contexto de Negócio
O frontend atual (src/) contém o início da galeria IA Videos. Vamos reorganizá-lo para o monorepo em `frontend/` e rebranding para Danuzio: troca da paleta violet para dourado/amber, tipografia mais cinematográfica, logo do Backstage. A estrutura de rotas precisa suportar: área pública (landing, login), área de membro protegida (/biblioteca, /roteiro/:slug) e área admin (/admin).

## Escopo

### Inclui
- Confirmar que o frontend está em `frontend/` com `vite.config.js` apontando `root: 'frontend'`
- Instalar dependências: `react-router-dom`, `axios` (para chamadas à API backend)
- Adaptar paleta Tailwind CSS v4: trocar `violet` por `amber/dourado`; manter fundo `#030406`, surface `#080a0e`
- Configurar variáveis de ambiente: `VITE_API_URL=http://localhost:8000`
- `AuthContext` + `useAuth` hook: armazena estado de sessão (email, isAuthenticated)
- `apiClient.js`: instância Axios configurada com `withCredentials: true` (cookie-based)
- Roteamento com React Router v6:
  - `/` → LandingPage (pública)
  - `/login` → LoginPage (pública)
  - `/auth/verificar` → MagicLinkCallbackPage (pública)
  - `/biblioteca` → BibliotecaPage (protegida)
  - `/roteiro/:slug` → RoteiroPage (protegida)
  - `/admin` → AdminLayout (protegida, role admin/operator)
- `ProtectedRoute` wrapper que redireciona para `/login` se não autenticado
- Componentes UI base: `Button`, `Badge`, `Tag`, `Spinner`, `EmptyState`
- Layout shell: `AppShell` (header com logo Backstage + botão sair)

### Não inclui (fica para outro plano)
- Conteúdo das páginas (BibliotecaPage, RoteiroPage) → Planos 7, 8
- Páginas de login (UI) → Plano 8
- Painel admin (UI) → Plano 9
