# Plano 9 – Frontend: Painel Administrativo

## Objetivo
Construir a interface do painel administrativo: login de admin, CRUD de roteiros com upload de arquivos, visualização de assinantes (via API Lastlink passada pelo backend) e audit log.

## Contexto de Negócio
O painel admin é de uso interno — Danuzio e sua equipe. A operação de publicar um novo roteiro deve levar 10 minutos. O admin faz login+senha+2FA, cria ou edita um roteiro, faz upload da capa e do PDF, define a data de publicação no Instagram (o backend calcula o `available_at` como 1h antes). Não há gerenciamento de assinaturas no admin — é somente leitura via Lastlink.

## Escopo

### Inclui

**Auth Admin:**
- `AdminLoginPage`: formulário email+senha → `POST /admin/login`
- `AdminTotpPage`: input de 6 dígitos TOTP → `POST /admin/2fa/verificar`
- `AdminProtectedRoute`: redireciona para `/admin/login` se não autenticado como admin

**Dashboard e Roteiros:**
- `AdminLayout`: sidebar com itens: Roteiros, Assinantes, Audit Log
- `AdminRoteirosPage`: lista de roteiros com filtro (publicados/agendados/rascunhos)
  - Tabela com colunas: título, status, data publicação Instagram, data disponibilização portal, ações
  - Botão "Novo roteiro", botões editar/despublicar por linha
- `RoteiroFormPage` (criar + editar): formulário com campos:
  - Título, slug (auto-gerado do título, editável)
  - Sinopse (textarea)
  - Temas (input de tags)
  - Upload de capa (imagem preview) → `POST /admin/roteiros/{id}/cover`
  - Upload de PDF → `POST /admin/roteiros/{id}/pdf`
  - Data/hora de publicação no Instagram (`published_at`)
  - Data/hora de disponibilização no portal (`available_at`) — sugestão automática: 1h antes
  - Botão salvar como rascunho e botão publicar (com confirmação)

**Assinantes:**
- `AdminAssinantesPage`: chama `GET /admin/assinantes` (backend proxia para Lastlink)
  - Lista somente leitura: email, plano, status, data de início
  - Filtros por status e plano
  - Link "Gerenciar no Lastlink" para cada assinante

**Audit Log:**
- `AdminAuditLogPage`: chama `GET /admin/audit-log`
  - Lista últimas 100 ações: admin, ação, alvo, data
  - Filtro por tipo de ação e data
