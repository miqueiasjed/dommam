# Plano 1 – Setup do Monorepo + Infraestrutura Laravel

## Objetivo
Configurar a estrutura monorepo (backend/ + frontend/), instalar o Laravel 12, criar todas as migrations do banco e preparar a base para os planos seguintes.

## Contexto de Negócio
O portal usa uma arquitetura monorepo onde o backend é uma API REST Laravel e o frontend é uma SPA React. O banco só armazena conteúdo e logs — nunca dados de assinantes (responsabilidade da Lastlink).

## Escopo

### Inclui
- Criar pasta `backend/` com Laravel 12 via `composer create-project`
- Mover código React atual de `src/` para `frontend/src/` (ajustar paths do vite.config.js)
- Configurar `.env.example` do backend (DB, Redis, mail, Lastlink secret, storage)
- Criar `BaseApiController` com helpers `success()`, `created()`, `error()`
- Criar todas as migrations: `roteiros`, `sessions`, `downloads`, `webhooks_log`, `admin_users`, `admin_audit_log`
- Configurar CORS no Laravel para aceitar o frontend (localhost dev + domínio prod)
- Configurar Sanctum para sessão httpOnly (cookie-based, não token Bearer)
- Configurar Storage (filesystem local ou S3) para capas e PDFs
- Configurar Redis para cache e queues

### Não inclui (fica para outro plano)
- Lógica de autenticação → Plano 2
- Models Eloquent e relacionamentos → Plano 2
- Qualquer endpoint de API → Planos 2+
- Estilização do frontend → Plano 6
