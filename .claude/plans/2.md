# Plano 2 – Autenticação por Magic Link (Backend)

## Objetivo
Implementar o sistema completo de autenticação de membros via magic link, validação de sessão e verificação de assinatura ativa na Lastlink.

## Contexto de Negócio
Membros não têm senha. O fluxo é: informam o e-mail → recebem link de login (expira em 15 min) → clicam → sessão criada (cookie httpOnly) → a cada request em rota protegida, consultamos a API Lastlink para confirmar assinatura ativa (cacheado por 5-15 min). Sem assinatura ativa → sessão encerrada + redirect com mensagem.

## Escopo

### Inclui
- Model `Sessao` (com scopes: `ativa()`, `expirada()`)
- `MagicLinkService`: gerar token, gravar em `sessions`, enviar e-mail via `EmailService`
- `SessaoService`: verificar token, criar sessão, revogar sessão, listar sessões ativas por e-mail
- `LastlinkService`: wrapper para `GET /api/v1/subscribers/{email}` com cache Redis (5-15 min)
- Middleware `VerificarAssinaturaAtiva`: valida cookie + consulta Lastlink
- `AuthController`: endpoints `POST /api/auth/solicitar-link` e `GET /api/auth/verificar`
- `LogoutController`: `POST /api/auth/logout` revoga sessão atual
- `FormRequest`: `SolicitarMagicLinkRequest` (valida e-mail), `VerificarTokenRequest`
- Testes básicos: token válido, token expirado, assinatura inativa

### Não inclui (fica para outro plano)
- Frontend do fluxo de login → Plano 8
- Admin auth (login+senha+2FA) → Plano 3
- Envio de e-mail de boas-vindas → Plano 4

## Dependências
